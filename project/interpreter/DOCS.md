# GraphLang Typechecker

## Обзор

`GraphLangTyper` — это тайпчекер для GraphLang, использующий паттерн Visiter. Он проходит по всему дереву и выводит типы отдельно для каждой его части, сохраняя переменные в специальный класс.

## Класс `GraphLangTyper`

- **`__init__`**: Инициализирует тайпчекер и создаёт объект `VariableStore` для хранения объявленных переменных.

---

### Методы

- **`printVariables()`**: Выводит все сохранённые на данный момент переменные, а также их типы.
- **`visitProg(ctx)`**, **`visitStmt(ctx)`**, **`visitDeclare(ctx)`**, **`visitAdd(ctx)`**, **`visitRemove(ctx)`**, **`visitBind(ctx)`**, **`visitExpr(ctx)`**, **`visitExpr(ctx)`**, **`visitRegexp(ctx)`**, **`visitSelect(ctx)`**, **`visitV_filter(ctx)`**: выводят тип для соответствующей части дерева.

---
- **`visitSet_expr(ctx)`**, **`visitEdge_expr(ctx)`**, **`__get_var_name(ctx)`**, **`visitRange(ctx)`**, **`visitNum(ctx)`**, **`visitChar(ctx)`**, **`visitVar(ctx)`**: специальные методы, выполняющие доп. проверки на данные. Например, проверяют список сохранённых переменных.

---

## Типы
- **`GRAPH`**: Граф.
- **`EDGE`**: Ребро графа.
- **`NUM`**: Число.
- **`CHAR`**: Символ.
- **`FA`**: Конечный автомат.
- **`RSM`**: Рекурсивный автомат.
- **`SET`**: Множество чисел.
- **`PAIR_SET`**: Множество пар чисел.
- **`RANGE`**: Диапазон.

---

## Обработка ошибок

Проверка типов вызывает исключения для следующих сценариев:
- **Необъявленные переменные**: Ссылка на переменную до объявления.
- **Несоответствие типов**: Выполнение операций с несовместимыми типами.
- **Недопустимые конструкции**: Использование недопустимых конструкций для операций (например, добавление ребра к переменной, не являющейся графом).

---

## Пример использования

```python
        types_visitor = GraphLangTyper()

        try:
            types_visitor.visit(tree)
        except Exception as ex:
            print(str(ex))
```

# GraphLang Runner

## Обзор

`GraphLang Runner` — это интерпретатор для GraphLang, использующий паттерн Visiter. Он проходит по всему дереву и исполняет отдельно каждую операцию.

---

### Класс GraphLangRunner

`GraphLangRunner` — это основной класс, который обрабатывает все операции. Он наследуется от `GraphVisitor`.

### Переменные

Переменные используются для хранения графов, конечных автоматов и других значений. Объявленные переменные сохраняются в словаре:

```python
self.__variables = {}
```

### Результаты запросов

Результаты запросов сохраняются отдельно, для более удобного доступа к ним при тестировании:

```python
self.__query_results = {}
```

Свойство `last_query_results` предоставляет копию самых последних результатов запросов.

### Регулярные выражения

Регулярные выражения представляются в виде `EpsilonNFA`. Если в регулярном выражении встречается переменная, она записывается как обычный символ, но заглавными буквами.

Когда появляется необходимость выполнить запрос `select`, автоматы объединяются в один `RecursiveAutomata`. В нём происходит подставновка автоматов вместо заглавных букв (нетерминалов). То есть подстановка происходит в момент, когда значения всех переменных уже известны.

### Запросы к графам

Метод `visitSelect` обрабатывает запросы к графам. Он:

1. Определяет стартовые и конечные узлы на основе фильтров (`FOR`).
2. Строит автомат запросов, используя функцию`build_rsm`.
3. Выполняет алгоритм CFPQ (`tensor_based_cfpq`) для получения результатов.


### Пример использования

```python
        runner_visitor = GraphLangRunner()

        try:
            types_visitor.visit(tree)
        except Exception as ex:
            print(str(ex))
```
